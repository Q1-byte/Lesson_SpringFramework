Dynamic Search Engine

MyBatis Dynamic SQL을 활용하여 제목, 내용, 작성자, 상품번호 등 다양한 검색 조건을 단일 쿼리로 처리합니다.

<sql id="criteria">
  <trim prefix="(" suffix=") AND " prefixOverrides="OR">
    <foreach item='type' collection="typeArr">
      <choose>
        <when test="type == 'T'">
          OR title LIKE CONCAT('%',#{keyword},'%')
        </when>
        <when test="type == 'C'">
          OR content LIKE CONCAT('%',#{keyword},'%')
        </when>
        <when test="type == 'N'">
          OR num = #{keyword}
        </when>
      </choose>
    </foreach>
  </trim>
</sql>

Secure File Upload

파일 업로드 시 보안과 중복 방지를 위해 UUID 기반 파일명과 이미지 미리보기를 구현했습니다.

// ProductController.java - File Upload
@PostMapping("/register")
public String register(ProductDTO dto, MultipartFile file) {
    if (!file.isEmpty()) {
        // UUID로 고유 파일명 생성
        String uuid = UUID.randomUUID().toString();
        String fileName = uuid + "_" + file.getOriginalFilename();
        
        // 서버에 파일 저장
        Path savePath = Paths.get(uploadPath, fileName);
        file.transferTo(savePath.toFile());
        
        dto.setImageFile(fileName);
    }
    productService.register(dto);
    return "redirect:/product/list";
}

Security Features

  • UUID Naming - 파일명 충돌 방지 및 원본명 노출 차단
  • MIME Type Check - 허용된 이미지 파일만 업로드
  • Image Preview - 업로드 전 클라이언트 미리보기
  • Path Traversal - 경로 조작 공격 방지

03. REST API & Reply System

@RestController · Fetch API · Asynchronous Communication Core Features II

RESTful Reply API

상품별 댓글(Reply) 기능을 REST API로 구현하여 페이지 새로고침 없이 비동기 CRUD가 가능합니다.

@RestController
@RequestMapping("/replies")
public class ReplyController {
    
    @PostMapping("/new")
    public ResponseEntity<String> create(@RequestBody ReplyDTO dto) {
        replyService.register(dto);
        return ResponseEntity.ok("success");
    }
    
    @GetMapping("/list/{pno}")
    public ResponseEntity<List<ReplyDTO>> getList(@PathVariable Long pno) {
        return ResponseEntity.ok(replyService.getList(pno));
    }
    
    @DeleteMapping("/{rno}")
    public ResponseEntity<String> remove(@PathVariable Long rno) {
        replyService.remove(rno);
        return ResponseEntity.ok("success");
    }
}

📡 HTTP Method Convention

POST(생성), GET(조회), PUT(수정), DELETE(삭제) RESTful 규약 준수

Client-Side Fetch API

JavaScript Fetch API를 활용한 비동기 통신으로 SPA 스타일의 사용자 경험을 제공합니다.

// Fetch API - 댓글 등록
async function addReply(reply) {
    const response = await fetch('/replies/new', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(reply)
    });
    
    if (!response.ok) throw new Error('Server Error');
    return await response.text();
}

// 댓글 목록 조회
async function getReplyList(pno) {
    const response = await fetch(`/replies/list/${pno}`);
    return await response.json();
}

Key Implementation

  • ResponseEntity - HTTP 상태 코드와 함께 응답 반환
  • @RequestBody - JSON 자동 파싱 (Jackson)
  • async/await - 가독성 높은 비동기 코드
  • Error Handling - 서버 오류 시 적절한 예외 처리

04. Spring Security Authentication

Role-Based Access Control · Custom UserDetails · Login Handler Security Implementation

Role-Based Access Control

관리자 전용 백오피스를 위해 Spring Security 6.5를 도입하여 역할 기반 접근 제어(RBAC)를 구현했습니다.

// CustomLoginSuccessHandler.java
@Override
public void onAuthenticationSuccess(HttpServletRequest request,
        HttpServletResponse response, Authentication auth) {
    
    boolean isAdmin = auth.getAuthorities().stream()
        .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"));
    
    if (isAdmin) {
        response.sendRedirect("/product/list");  // 관리자
    } else {
        response.sendRedirect("/accessDenied");  // 일반 사용자
    }
}

Security Components

  • CustomUserDetailsService - DB 회원 정보를 UserDetails로 변환
  • CustomUser - Security User 확장, MemberDTO 포함
  • CustomAccessDeniedHandler - 403 에러 커스텀 처리
  • CustomLoginSuccessHandler - 역할별 로그인 후 리다이렉트

AOP Logging

Cross-Cutting Concern 분리

Service 계층 메서드 실행 시 자동 로깅으로 비즈니스 로직과 로깅 관심사 분리

@Before - 파라미터 로깅

@Before("execution(* org.zerock.service.*.*(..))")
public void logParams(JoinPoint jp) {
    Object[] params = jp.getArgs();
    log.info(Arrays.toString(params));
    
    Object target = jp.getTarget();
    log.info("target : " + target);
}

@Around - 실행 시간 측정

@Around("execution(* org.zerock.service.*.*(..))")
public Object logTime(ProceedingJoinPoint pjp) 
    throws Throwable {
    
    long start = System.currentTimeMillis();
    Object result = pjp.proceed();
    long end = System.currentTimeMillis();
    
    log.info("TIME : " + (end - start) + "ms");
    log.info("Method : " + pjp.getSignature());
    return result;
}

AOP 장점

  • 코드 중복 제거 - 각 메서드에 로깅 코드 불필요
  • 유지보수성 - 로깅 정책 한 곳에서 관리
  • 성능 모니터링 - 메서드 실행 시간 자동 측정

06. Challenges & Roadmap

Problem Solving · Troubleshooting · Future Plans Project Retrospective

Engineering Challenges

🔴 N+1 Query Problem

문제: 목록 조회 시 각 항목마다 추가 쿼리 실행으로 성능 저하
해결: 반정규화 컬럼 추가 + 트랜잭션 동기화로 O(N) → O(1) 개선

🟠 Transaction Consistency

문제: 게시글/파일 저장 중 일부 실패 시 데이터 불일치
해결: @Transactional로 전체 롤백 처리, ACID 보장

🟢 Security Context Issue

문제: MvcRequestMatcher의 ApplicationContext 분리 에러
해결: request-matcher="ant" 설정으로 AntPathMatcher 사용

Key Learnings

  • 계층 분리 - Controller/Service/Mapper 책임 명확화
  • 보안 설계 - 인증/인가 분리 및 역할 기반 접근 제어
  • 트러블슈팅 - 에러 로그 분석 및 단계적 디버깅
